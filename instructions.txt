Role: You are a senior web developer. Your task is to build a high-performance, single-page holiday rental website for a property in Belgium. The site must support a "Semi-Direct" booking strategy in French.

Technical Constraints:

    SSG: Jekyll.

Styling: Tailwind CSS (CDN).

Icons: Lucide Icons (CDN).

Hosting/Backend: Cloudflare Pages with Cloudflare Pages Functions.

    Database: Cloudflare D1 (Multi-tenant architecture using a single shared database for all properties).

    Payments: Stripe Checkout / Stripe Connect API (or Mollie).

Requirement 1: Content & Data Structure

    Create a flexible system to display property information. This data must be easy to edit (e.g., via a YAML file) and should include:

    Property Name and Tagline.

A main Description area (placeholder for now).

Technical details: Address, Registration Number (Numéro d'enregistrement), and a list of amenities/features.

The site must include a placeholder for a photo gallery that can dynamically load images from a local project folder.

Requirement 2: Availability Calendar

    The site must display an availability calendar.

The calendar logic must rely on an Airbnb iCal URL to fetch and display "booked" vs. "available" dates.

The UI should clearly distinguish between these two states to the user.

Requirement 3: Booking Inquiry Logic & Database

    Provide a functional inquiry form including fields for: Check-in/Check-out dates, Number of guests, Name, Email, and Message.

Include a mandatory checkbox for the user to agree to the "Conditions Générales de Vente" (Rental Terms).

Backend Integration: The form must submit data to a Cloudflare Pages Function.

Database Logic: The function must save the inquiry to the Cloudflare D1 database. To ensure GDPR compliance and multi-tenant security, the SQL insertion must enforce a property_id (tenant ID) and set the initial status to "Pending". Do not store any sensitive financial data in this database.

Success State: Upon successful submission, the UI must replace the form with a confirmation message explaining that the owner will review the request and send a secure payment link (Bancontact/Card) shortly.

Requirement 4: The Payment & Approval Ecosystem

    Approval Flow: Design the Cloudflare Function logic to send the owner an email notification containing an "Approve" action/endpoint.

    Checkout Generation: Upon owner approval, a function must trigger the Stripe/Mollie API to generate a secure Checkout Session URL and email it to the guest. This implementation should support platform fee routing (e.g., Stripe Connect) so a percentage can be collected.

    Webhook Handling: Provide a webhook handler function that listens for successful payments. It must update the specific booking status to "Paid" in the D1 database (querying securely using the property_id) and trigger a final confirmation email to both parties.

Requirement 5: Belgian Compliance & Trust

    The layout must provide space to display the mandatory Walloon Registration Number and professional insurance credentials (e.g., "Lodge Protect" or "Abandon de Recours").

The language of the entire interface must be French.
